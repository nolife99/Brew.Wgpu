# Brew.Wgpu

Pure-managed C# bindings for [wgpu-native](https://github.com/gfx-rs/wgpu-native) (WebGPU), with the
native binaries bundled per-RID. Consumers need **no native build step** — `DllImport("wgpu_native")`
resolves the correct `runtimes/<rid>/native/` library at run time.

## Install

```
dotnet add package Brew.Wgpu
```

## What's in the package

- **`Brew.Wgpu.Native`** — raw P/Invoke bindings generated by ClangSharp from the wgpu-native headers
  (opaque handle structs, `delegate* unmanaged[Cdecl]` callbacks, blittable structs, `DisableRuntimeMarshalling`).
- **`Brew.Wgpu`** — a thin, safe wrapper: `Instance`, `Adapter`, `Device`, `Queue`, `Surface`,
  buffers/textures/views/samplers, bind groups, pipelines, and command/render/compute encoders.
- **Double-free-safe resource handles** — resource handles are generational value types
  (`{slot, generation}`) backed by a per-kind `HandleTable`. Copies are free; disposal releases exactly
  once and invalidates every copy, so a double-dispose is a no-op rather than a use-after-free. Scope-bound
  encoders remain `ref struct`s.
- **`runtimes/<rid>/native/`** — wgpu-native binaries for each supported RID (added by CI at publish time).

## Supported runtimes

`win-x64`, `win-arm64`, `linux-x64`, `linux-arm64`, `osx-x64`, `osx-arm64` by default. 32-bit Windows
(`win-x86`) can be added by enabling the `i686-pc-windows-msvc` row in the native build matrix — the
binding types are already correct for it.
